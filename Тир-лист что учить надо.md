```
✅ Базовая безопасность: security groups, ключи, шифрование на rest

[ ] Могу загрузить файл в S3 и выдать публичную ссылку
[ ] Знаю, как создать IAM-пользователя с ограниченными правами
[ ] Понимаю разницу между region, availability zone, edge location
```

```hcl
# Обязательно:
```

```
✅ Обновления: регулярный патчинг ОС, обновление базовых образов

# Чек-лист:

[ ] Понимаю, почему нельзя запускать контейнеры от root без необходимости
[ ] Знаю, как ротировать секреты и где их хранить (Vault / облачные SM)
```

> 💡 **Итог по A-тиру:** Это тот минимум, который ожидают от кандидата на позицию Junior+/Middle. Если S-тир — это "как работать", то A-тир — это "как работать хорошо и безопасно".

### 🔹 Базы данных для DevOps
```

✅ Миграции: инструменты (Flyway, Liquibase), идемпотентность, откаты

```

```
✅ Отказоустойчивость: retry, circuit breaker, bulkhead, timeout

✅ Балансировка: L4 vs L7, health checks, sticky sessions

# Чек-лист:
```

> 💡 **Итог по B-тиру:** Эти темы ты начнёшь встречать в реальных задачах. Не учи их "впрок" — разбирай по мере необходимости, но держи в списке приоритетов.

## 🟠 C-ТИР: Специализация (учи, когда определился с направлением)

### 🔹 Углублённая безопасность
```
• Policy as Code: OPA/Gatekeeper, Kyverno для Kubernetes
• Secrets management: HashiCorp Vault (динамические секреты, PKI)
• Compliance: SOC2, ISO27001, GDPR — как технически обеспечивать
• Runtime security: Falco, Tetragon для детектирования аномалий
```

### 🔹 Продвинутый мониторинг и SRE
```
• Distributed tracing: OpenTelemetry, Jaeger, Tempo
• SLO/SLI/SLA: как определять, измерять и использовать в алертинге
• Error budget: как принимать решения о релизах на основе бюджета ошибок
• Chaos Engineering: принципы, инструменты (Chaos Mesh, Litmus)
```

### 🔹 Мульти-кластер и гибридные окружения
```
• Управление множеством кластеров: Cluster API, Rancher, GitOps-паттерны
• Service Mesh: Istio, Linkerd — когда действительно нужно
• Edge / IoT: специфика деплоя и управления на периферии
• Hybrid cloud: синхронизация между on-prem и облаком
```

### 🔹 Платформенная инженерия
```
• Internal Developer Platform: Backstage, Humanitec, Cortex
• Self-service инфраструктура: Crossplane, Terraform Cloud/Enterprise
• Golden paths: как создавать шаблоны, которые хотят использовать разработчики
• Developer experience: метрики и фидбек-лупы для улучшения платформы
```

> 💡 **Итог по C-тиру:** Это уже специализация. Не пытайся выучить всё — выбери 1-2 направления, которые интересны или востребованы в твоей компании.

```
📅 Месяцы 1-3: Фокус на S-тир
  • Ежедневная практика: Linux, Git, Bash
  • Проект: развернуть приложение в Docker, настроить простой CI
```
